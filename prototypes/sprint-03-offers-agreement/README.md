# Sprint 3 — Offers & Agreement Flow

هذا الـ Sprint يغطي مرحلة العروض والاتفاق داخل منصة محايد.

## الهدف

تحويل طلب المشروع إلى عرض واضح قابل للمراجعة والقبول، ثم إنشاء اتفاق موثق بين العميل ومقدم الخدمة قبل بداية التنفيذ.

## الشاشات الموجودة

1. تفاصيل طلب المشروع لمقدم الخدمة
2. قبول أو رفض الدعوة
3. إنشاء عرض
4. تحديد نطاق العمل
5. تقسيم العرض إلى مراحل
6. معاينة العرض قبل الإرسال
7. تم إرسال العرض
8. قائمة العروض المستلمة للعميل
9. تفاصيل العرض
10. طلب تعديل على العرض
11. ملخص الاتفاق
12. انتظار موافقة الطرف الآخر
13. تم توثيق الاتفاق
