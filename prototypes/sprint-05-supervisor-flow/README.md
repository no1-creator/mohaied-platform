# Sprint 5 — Supervisor Flow

هذا الـ Sprint يغطي دور المشرف المتخصص داخل منصة محايد.

## الهدف

إضافة طرف محايد خبير يراجع مراحل المشروع والتسليمات فنيًا، ويكتب تقارير موثقة تساعد في اتخاذ القرار وتقليل النزاعات بين العميل ومقدم الخدمة.

## الشاشات الموجودة

1. ما دور المشرف
2. قائمة المشرفين المرشحين
3. ملف المشرف
4. تأكيد اختيار المشرف
5. تم إرسال الدعوة
6. دعوة إشراف للمشرف
7. لوحة المشرف
8. مراجعة الاتفاق
9. مراجعة تسليم
10. كتابة تقرير المشرف
11. قرار المشرف
12. تم إرسال التقرير

## ملاحظات Product

- المشرف اختياري، ويمكن إضافته قبل أو أثناء المشروع.
- المشرف يجب أن يقبل التكليف قبل أن يبدأ المراجعة.
- كل مراجعة للمشرف يجب أن ترتبط بمرحلة محددة.
- تقرير المشرف يجب أن يظهر للطرفين.
- قرار المشرف استشاري في MVP، والاعتماد النهائي يبقى للعميل.
- تكلفة المشرف تحسب لكل مرحلة يتم مراجعتها.
- المشرف لا يملك صلاحية تعديل التسليمات، فقط المراجعة والتقرير.

## حالات تكليف المشرف

- Invited
- Accepted
- Declined
- Active
- Completed
- Removed

## حالات تقرير المشرف

- Draft
- Submitted
- Approved Delivery
- Needs Revision
- Needs Clarification

## ملاحظات للمطور

- Supervisor مرتبط بـ Project عبر Assignment.
- كل Assignment له حالة وتكلفة.
- Supervisor Report مرتبط بـ Milestone و Submission.
- يجب حفظ كل تقرير كنسخة مستقلة للرجوع إليها.
- قرار المشرف يسجل في Activity Log.
- يجب فصل صلاحيات المشرف عن صلاحيات العميل ومقدم الخدمة.
